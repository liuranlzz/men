# 剪映草稿生成工具使用说明

目标产物：一个可以在剪映中打开的草稿，里面包含 AI 配音音轨和可控样式的字幕轨。

## 推荐使用形态

第一版建议做成一个命令行工具，对外只有一个主入口：

```bash
python tools/jianying_draft/create_draft.py \
  --input "content/遥控4.0/遥控4.0内容.md" \
  --config "config/jianying_draft.yaml"
```

工程内部可以拆成多个模块，但用户使用时不应该关心这些细节：

```text
tools/jianying_draft/
  create_draft.py        # 主入口
  parse_script_md.py     # 解析 Markdown 口稿
  ai_structurer.py       # 可选：用 DeepSeek 把口稿整理成标准 JSON
  tts.py                 # 生成配音音频
  subtitle.py            # 生成 SRT 字幕
  draft_writer.py        # 写入剪映草稿
  schemas.py             # 中间数据结构
```

也可以先做成单文件脚本验证 POC：

```text
tools/jianying_draft/create_draft.py
```

但建议脚本内部保留清晰函数边界，等跑通后再拆模块。

## 入参

### 1. 必需入参：口稿 Markdown

示例：

```bash
--input "content/遥控4.0/遥控4.0内容.md"
```

当前项目里的口稿格式已经适合解析：

```text
#### 【0-6秒】 黄金开头：痛点展示

- **画面**：实拍手部特写...
- **知性女声**：
    “新配的卷帘门遥控器，按键功能全乱了？千万别扔！...”
```

工具会提取：

- 时间段：`0-6秒`
- 画面说明：作为视频占位提示或后续分镜信息
- 配音角色：例如 `知性女声`
- 口播文本：生成 TTS 音频
- 字幕文本：默认等于口播文本

### 2. 必需入参：配置文件

示例：

```bash
--config "config/jianying_draft.yaml"
```

建议配置内容：

```yaml
jianying_draft_root: "/Users/xuantu/Movies/JianyingPro/User Data/Projects/com.lveditor.draft"

canvas:
  width: 1080
  height: 1920

subtitle:
  size: 8.0
  transform_y: -0.78
  color: [1.0, 1.0, 1.0]
  border_color: [0.0, 0.0, 0.0]
  border_width: 45.0
  shadow_alpha: 0.35
  max_line_width: 0.86

tts:
  provider: edge
  voice: "zh-CN-XiaoxiaoNeural"
  rate: "+0%"
  volume: "+0%"

ai:
  provider: deepseek
  model: deepseek-v4-flash
  use_for: ["structure", "caption_split"]
```

### 3. 可选入参：运行模式

建议支持：

```bash
--dry-run
```

只生成结构化 JSON、字幕和任务清单，不写入剪映草稿。

```bash
--input-format md
```

输入是项目口稿 Markdown。

```bash
--input-format manifest
```

输入是已经整理好的中间 JSON。

```bash
--input-format coze
```

输入是 Coze 工作流输出 JSON。

```bash
--draft-name "遥控4.0-自动草稿"
```

指定剪映草稿名称。

## 中间数据

工具应先把口稿转成统一 manifest，再生成草稿。这样后面接 Coze、DeepSeek 或其他工作流都容易。

示例：

```json

  "project_name": "遥控4.0",
  "canvas": {
    "width": 1080,

  "segments": [
```

## 产物

一次运行建议产生两类产物。

### 1. 工作区产物

放在项目内，便于调试和复用：

```text
generated/jianying_draft/遥控4.0-自动草稿/
  manifest.json
  captions.srt
  run.log
  audio/
    audio_000.mp3
    audio_001.mp3
  assets/
    placeholder.png
```

这些产物用于排查问题：

- `manifest.json`：口稿解析结果
- `captions.srt`：导入剪映的字幕
- `audio/*.mp3`：AI 配音音频
- `run.log`：草稿目录、耗时、失败原因等

### 2. 剪映草稿产物

最终写入剪映草稿目录：

```text
/Users/xuantu/Movies/JianyingPro/User Data/Projects/com.lveditor.draft/遥控4.0-自动草稿/
  draft_content.json
  draft_info.json
  timeline_layout.json
  materials/
  ...
```

验收标准：

- 打开剪映后能看到草稿
- 草稿能进入时间线
- 有音频轨
- 有字幕轨
- 字幕样式符合配置
- 修改配置重新生成后，字幕样式能变化

## DeepSeek API 能否用上

可以用，但它不是配音工具。

DeepSeek 官方 API 文档显示它提供 OpenAI/Anthropic 兼容接口，主要通过 `https://api.deepseek.com/chat/completions` 调用文本模型，例如 `deepseek-v4-flash`、`deepseek-v4-pro`。官方文档的 API Reference 目前列出的是 Chat、Completions、Models 等文本接口，没有看到官方 TTS/语音合成接口。

所以 DeepSeek 在这个工具里适合做：

- 把不规整的 Markdown 口稿整理成标准 JSON
- 修复或补全缺失时间轴
- 把长口播拆成更适合字幕的短句
- 生成每段画面占位提示词
- 生成 Coze 工作流所需 JSON
- 检查字幕是否过长、是否需要换行

DeepSeek 不适合做：

- 直接生成 mp3/wav 配音文件
- 调用剪映内置 AI 配音
- 写入本地剪映草稿目录

## DeepSeek 使用方式

环境变量：

```bash
export DEEPSEEK_API_KEY="你的 key"
```

命令：

```bash
python tools/jianying_draft/create_draft.py \
  --input "content/遥控4.0/遥控4.0内容.md" \
  --config "config/jianying_draft.yaml" \
  --ai-provider deepseek
```

在代码里可以使用 OpenAI SDK 兼容调用：

```python
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["DEEPSEEK_API_KEY"],
    base_url="https://api.deepseek.com",
)

response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[
        {"role": "system", "content": "你负责把短视频口稿整理成严格 JSON。"},
        {"role": "user", "content": markdown_text},
    ],
    response_format={"type": "json_object"},
    extra_body={"thinking": {"type": "disabled"}},
)
```

注意：DeepSeek 只负责输出文本 JSON。配音音频仍需要 TTS provider 生成。

## 推荐第一版命令设计

最小可用：

```bash
python tools/jianying_draft/create_draft.py \
  --input "content/遥控4.0/遥控4.0内容.md" \
  --config "config/jianying_draft.yaml"
```

只解析不生成草稿：

```bash
python tools/jianying_draft/create_draft.py \
  --input "content/遥控4.0/遥控4.0内容.md" \
  --config "config/jianying_draft.yaml" \
  --dry-run
```

使用 DeepSeek 整理口稿：

```bash
DEEPSEEK_API_KEY="你的 key" \
python tools/jianying_draft/create_draft.py \
  --input "content/遥控4.0/遥控4.0内容.md" \
  --config "config/jianying_draft.yaml" \
  --ai-provider deepseek
```

使用 Coze 输出：

```bash
python tools/jianying_draft/create_draft.py \
  --input "coze_output.json" \
  --input-format coze \
  --config "config/jianying_draft.yaml"
```

## 推荐实现顺序

1. 不接 DeepSeek，先用规则解析 `遥控4.0内容.md`
2. 生成 `manifest.json`
3. 生成 `captions.srt`
4. 生成或接入 TTS 音频
5. 用 `pyJianYingDraft` 写入草稿
6. 在剪映中校准字幕样式
7. 再接 DeepSeek 做口稿整理和字幕拆句
8. 最后再接 Coze JSON

这个顺序最稳：先把“剪映草稿可打开”打穿，再逐步引入 AI 编排。
